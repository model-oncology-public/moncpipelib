# moncpipelib: What's New Since v0.7.0

*Covers v0.7.1 through v0.11.3 (~19,600 lines added across 78 files)*

## Breaking Changes to Call Out

1. **`pipeline_id` is now required** in all contract YAML files.
2. **`WriteMode.PARTITIONED` removed** -- use partition-aware `full_refresh`/`upsert`/`scd2` instead.
3. **`generate_asset_check()` deprecated** -- use `make_contract_checks()`.
4. **PII defaults to `true`** -- any contract column without explicit `pii: false` is treated as PII.

## Cookbook (Executable Docs)

- **`docs/cookbook.md`** is now auto-generated from annotated pytest tests in `tests/cookbook/`. CI regenerates it on every push. There are currently examples for: write modes (upsert, SCD2, full_refresh), batched streaming, partition-aware writes, contract checks, PII masking, OpenLineage events, and diagnostics.

## Data Contracts -- Much Stronger Enforcement

- **Contract sink mode enforcement.** The contract's sink `mode` field is now reconciled against the IO manager write mode at write time. Mismatches raise `ContractViolationError` before any data is written.
- **Contract primary key reconciliation.** Columns marked `primary_key: true` in the contract are auto-applied to upsert keys. Conflicting metadata vs. contract PKs raise errors.
- **`pipeline_id` (required).** A stable UUID that persists across asset renames so lineage history stays correlated. All contracts must now include this field.
- **PII column tracking.** New `pii` boolean field on contract columns (defaults to `true` -- safe by default). Drives PII masking in rendering, PII comments synced to PostgreSQL `COMMENT ON COLUMN`, and a new `ColumnClassificationFacet` in OpenLineage.
- **Dagster UI integration.** Contract validation results surface as materialization metadata (`contract_status`, check counts, violations). New `make_contract_checks()` auto-generates Dagster asset checks from contracts.
- **Implicit discovery deprecated.** Stack-frame introspection and cwd fallback for finding contracts now emit `DeprecationWarning`. Use explicit `search_paths`.

## Write Modes -- Partition-Aware Everything

- **Any write mode now supports partitions.** Instead of a standalone `PARTITIONED` mode, `full_refresh`, `upsert`, and `scd2` all gain automatic partition scoping when the Dagster context has partition keys and `partition_column` is configured. Guard rails raise `ContractViolationError` for unsafe combos (e.g. full_refresh + partitioned without a `partition_column`).
- **Batched SCD2 support.** `BatchedDataFrame` now works with SCD2 -- each batch is staged, then change detection runs once after all batches are consumed. Previously this combination was rejected.
- **SCD2 `detect_deletes`.** Opt-in parameter for full-snapshot pipelines that expires current records whose business key is absent from the incoming DataFrame. Empty DataFrames raise `ValueError` to prevent accidental full-table expiration.

## Universal Schema Routing (Migration 005)

- **Single IO manager, all schemas.** A single `PostgresIOManager` instance can now serve bronze/silver/gold schemas. Per-asset `target_schema` metadata or contract sinks drive routing via a resolution cascade: contract sink > `target_schema` metadata > `schema_override` > `default_schema` > `db_schema`.
- Layer (bronze/silver/gold) is derived automatically from the schema name.

## PII-Safe Rendering

- **`polars_to_md()`** -- new utility that converts Polars DataFrames to markdown tables with automatic PII masking when a contract is provided. Designed for safe logging in HIPAA environments.

## Lineage and UUID Handling

- **`PostgresPolarsSchema`** -- deterministic OID-to-Polars dtype mapping so UUID columns (including `_lineage_id`) are always read as `pl.String` instead of `pl.Object`.
- **psycopg2 UUID adapter** registered on all read paths so UUIDs come back as plain strings before Polars sees them.
- **`processed_at` migrated to `TIMESTAMPTZ`.**

## Diagnostics

- **`PodResourceSampler`** -- context manager that samples pod CPU/memory from cgroups v2 at configurable intervals, with structured JSON log output and Dagster metadata integration.

## Internal / Architecture

- `postgres.py` decomposed into focused modules (`io_managers/writers.py`, `io_managers/enums.py`, `contracts/reconciliation.py`).
- Writer thin wrappers removed; batched handler logic extracted cleanly.
